# Release Process (ReactorOps)

## Core Compatibility
Pin adopted DiamondOps-Core version here:
- Core compatibility: v0.1.x

## Versioning
- PATCH: doc fixes, clarifications
- MINOR: new service SKUs, additive checklists, new reporting categories
- MAJOR: breaking changes to service definitions or telemetry expectations

## Release steps
1. Update service docs/runbooks
2. Ensure telemetry docs reference Core schemas
3. Tag a GitHub Release `vX.Y.Z`
